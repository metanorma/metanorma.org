# Domain Glossary

Terms used throughout the codebase. Maintain alphabetical order.

## AsciiDoc
The source markup language for all Metanorma documentation. Files use
the `.adoc` extension. Parsed by **coradoc**, not asciidoctor.

## Coradoc
The approved AsciiDoc parser (`coradoc-adoc` gem). Converts `.adoc`
sources into a **CoreModel** document tree, then into **mirror-json**
envelopes. Installed via the project Gemfile — no local checkout required.

## CoreModel
Coradoc's canonical, format-agnostic document model. The intermediate
representation between source formats (AsciiDoc, Markdown, HTML) and
output formats (mirror-json, HTML, Markdown).

## Flavor
An organization-specific implementation of Metanorma (e.g., ISO, IEC,
IEEE). Each flavor has a gem (`metanorma-iso`), a repo, and a set of
supported deliverables. Catalog entries are in `src/data/flavors.ts`.

## Hub Page
A synthesized index page for a directory that has child pages but no
explicit `index.adoc` source. Generated by the converter's hub-pass.

## Mirror-JSON
A ProseMirror-style AST serialized as JSON, produced by coradoc from
`.adoc` sources. Each envelope contains `source`, `frontmatter`,
`title`, and `mirror_json` (the AST string). Stored in `mirror-json/`.

## Mirror Renderer
The TypeScript module (`src/lib/mirror-renderer.ts`) that converts a
mirror-json AST into HTML at build time. Uses an OCP-compliant registry
of node-type renderers.

## Pagefind
The client-side search engine. Indexes the built `dist/` directory
after the Astro build. Loaded at runtime via `<script>` tag injection
in `SearchButton.vue`.

## Software Entry
A gem or tool in the Metanorma toolchain. Categorized as `core`,
`cli`, `flavor`, or `tool`. Catalog entries are in `src/data/software.ts`.

## Sidebar Registry
The module (`src/config/registry.ts`) that resolves which sidebar items
to show for a given pathname. Merges hand-authored and auto-generated
sidebars with a precedence rule: hand-authored entries suppress
generated entries on both exact-key and sub-path conflicts.

## Shiki
Build-time syntax highlighter. Highlights `<pre><code>` blocks in the
mirror-loader at build time. Dual-theme output (github-light /
github-dark) that switches via the `.dark` class on `<html>`.
